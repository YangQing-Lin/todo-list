# Go Todo List 前端页面需求文档

## 项目概述

**项目名称**: Go Todo List Frontend
**技术栈**: React + TypeScript + Vite
**后端API**: http://localhost:7789
**设计理念**: 简洁、高效、响应式

## 页面结构设计

### 1. 主页面布局 (AppLayout)
**文件位置**: `src/components/AppLayout.tsx`
**功能描述**:
- 顶部导航栏（应用标题）
- 主内容区域
- 响应式布局
- 全局样式

### 2. Todo列表页面 (TodoPage)
**文件位置**: `src/pages/TodoPage.tsx`
**功能描述**:
- 展示所有Todo项目
- 添加新Todo功能
- 标记完成/未完成
- 删除Todo
- 筛选功能（全部/待办/已完成）

**UI组件需求**:
- Todo输入框
- Todo列表容器
- 筛选标签
- 空状态提示

### 3. Todo项目组件 (TodoItem)
**文件位置**: `src/components/TodoItem.tsx`
**功能描述**:
- 显示Todo内容
- 复选框（完成状态）
- 编辑功能
- 删除功能
- 显示创建时间

**状态处理**:
- hover效果
- 编辑模式切换
- 删除确认

## API接口对接

### 基础接口
```typescript
// 获取所有Todo
GET /api/todos

// 创建Todo
POST /api/todos
{
  "title": string,
  "description": string
}

// 更新Todo（预留）
PUT /api/todos/{id}

// 删除Todo（预留）
DELETE /api/todos/{id}
```

### API服务模块
**文件位置**: `src/services/api.ts`
**功能**:
- 封装所有API调用
- 错误处理
- 请求拦截器
- 响应数据格式化

## 组件设计规范

### 命名规范
- 组件文件：PascalCase (TodoItem.tsx)
- 变量名：camelCase (todoList)
- CSS类名：kebab-case (todo-item)

### Props接口定义
```typescript
interface TodoItemProps {
  todo: Todo;
  onUpdate: (id: number, updates: Partial<Todo>) => void;
  onDelete: (id: number) => void;
}
```

### 状态管理
- 使用React Hooks (useState, useEffect)
- Context API用于全局状态
- 本地存储持久化
